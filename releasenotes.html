Linda 2026 · Release 1

Released January 14, 2026 — 2026.1

New Features

Mobile

  • Linda Mobile 2026 (Beta): Smartphone-optimiert, reduzierte UI, kürzere Ladezeiten.
  • Separate mobile Ansicht/URL mit identischer Antwortlogik.

Efficiency

  • Token-/Payload-Optimierung: Kontexte kürzen, effizientere Requests.
  • UI-Regel: maximal 4 History-Einträge anzeigen.

Resolved Issues

Performance

  • Antworten kommen schneller durch optimierte Übergaben und reduzierte Payloads.

Model Usage

  • Teilweise Modernisierung der Modellnutzung (z. B. GPT-5.1 je nach Setup) für Qualität & Tempo.

Security Notes

Proxy Architecture

  • Client ruft keinen Make-Webhook direkt auf, sondern über einen Vercel-Proxy (Endpoint-Abschirmung).

Data Minimization

  • So wenig Kontext wie nötig senden; keine sensiblen Daten.
Status: „Lernen“ & „Fachmodus“ wurden angekündigt, sind aber (Stand Jan 2026) noch nicht final ausgerollt.

Linda Secure · Hardening Sprint

Released January 07–13, 2026 — Security Notes

Security Notes

Prompt Injection

  • Prompt-Injection ist inhaltliche Sicherheit und nicht rein durch HTML/JS lösbar.
  • Risiko: „System-Prefix + User-Text als ein String“ → Guardrails/Policy nötig.

Proxy & Observability

  • Client → Vercel Proxy → Make/Backend statt direkter Webhook-Calls.
  • Logging/Debugging als Standard, wenn „Make kommt nichts an“.

Resolved Issues

Proxy Stabilisierung

  • Saubere Request-Formate (Method/Headers/Body) und Timeouts/Fehlerbehandlung.

Testing

  • Red-Team-Tests (gezielt bösartige Inputs) als Qualitätscheck.

Known Issues

  • Prompt-Injection-Schutz benötigt serverseitige Guardrails (Policy, Rollen-Trennung, Tool-Constraints).

Governance · Prompt-Standard

Released June 2025 — Quality & Compliance

New Features

Antwort-Standard

  • Strukturierte Antworten im Corporate Design; klare Antwortlogik.
  • Quellen am Textende im Stil „Dokumentname, Seite, Kapitel“ (ggf. Fußnoten).

Vector Store & Quiz

  • Vector-Store-Nutzung verpflichtend, wenn verfügbar.
  • Quiz-Modus geschützt (Kennwort „Norwegen“).

Security Notes

  • DSGVO-Fokus: Datenminimierung und saubere Governance.
  • Keine technischen Artefakte/IDs in Antworten.
